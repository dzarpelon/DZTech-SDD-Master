# Framework Principles

The framework operates using these core principles:

- `spec-first`: repository changes follow defined specifications
- `human-final-authority`: human approval gates are authoritative
- `no-assumptions`: ambiguity requires stop-and-clarify behavior
- `fast-functional-delivery`: implement the minimal functional system required
- `centralized-method-distributed-execution`: the method is centralized, execution can vary by project
- `upgrade-by-decision-not-drift`: framework changes occur through explicit updates
