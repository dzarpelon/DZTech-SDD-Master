# Framework

This directory stores framework-level references that make the repository operational as a structured implementation.

Included artifacts:

- `manifest.md`: canonical framework identity, versioning, and file registry
- `principles.md`: core operating principles
- `ideation-to-spec-method.md`: idea-to-spec method
- `spec-canon.md`: canonical source-of-truth file set
- `workflow-state-model.md`: workflow states and transitions
- `validation-gates.md`: mandatory human approval gates
- `agent-execution-rules.md`: AI execution rules derived from repository governance
- `traceability.md`: mapping between workflow stages, actors, and repository locations
- `version.md`: framework version reference
